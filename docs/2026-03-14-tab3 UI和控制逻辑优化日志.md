# 2026-3-14 Tab3 UI和控制逻辑优化日志

## 背景

在完成 `Tab3 DAS` 基本功能联调后，发现界面和控制逻辑仍有几个明显问题：

- 左侧参数区布局过于拥挤，阅读和操作效率较低
- `space-time` 图缺少颜色条、色图切换和手动 `vmin/vmax` 控制
- `DAS monitoring` 按钮缺少明确的状态反馈和更强的视觉提示
- 缺少“暂停绘图但不断开通信”的独立控制
- 联合存储仍使用复选框，不利于快速判断开关状态

本次优化围绕以上问题，对 `Tab3` 的 UI 布局、空间时间图显示控制和按钮交互做了集中调整。

## 优化项一：左侧参数区布局压缩

### 存在问题

原来的 `Tab3` 左侧参数区按单列堆叠方式排布，几个区域占用行数较多：

- `IP` 和 `Port` 各占一行
- `Live Header` 四个字段各占一行
- `Channel Start / Channel End` 各占一行
- `Time Downsample / Space Downsample` 各占一行

结果是：

- 区域高度被拉长
- 参数密度偏低
- 需要频繁上下滚动视线
- 重要参数之间不便于横向比较

### 解决方案

对 `Tab3` 左侧参数区做了紧凑排版：

1. `IP` 与 `Port` 调整为同一行两列
2. `Live Header` 四个字段调整为两行，每行两个字段
3. `Live Header` 的名称和值之间横向间距缩小
4. `Channel Start / Channel End` 调整为一行
5. `Time Downsample / Space Downsample` 调整为一行

### 优化效果

- 左侧参数区更紧凑
- 信息密度更合理
- 常用参数可以在同一视野内完成查看和修改

## 优化项二：Space-Time 图增加颜色控制和颜色条

### 存在问题

原来的 `space-time` 图虽然能显示图像，但仍缺少常见的二维数据可视化控制能力：

- 无颜色条类型切换
- 无 `vmin/vmax` 手动输入框
- 无带数据统计分布的纵向颜色条

这会导致：

- 图像对比度不易人工调节
- 不同数据幅值下显示效果不稳定
- 无法快速观察当前图像强度分布

### 参考实现

本次参考了以下工程中 `time_space_plot.py` 的实现思路：

- `D:\OneDrive - EVER\00_KY\DAS-器件测试、上位机开发\BX\PCIe-7821\pcie7821_gui`

重点借鉴了两部分：

1. 使用 `ImageItem.setColorMap()` 切换色图
2. 使用 `HistogramLUTWidget` 作为带直方图分布的纵向颜色条

### 解决方案

本次在 `Tab3` 的 `space-time` 区域新增了以下能力：

1. 在图上方增加 `Colormap` 下拉框
2. 新增 `Vmin` 输入框
3. 新增 `Vmax` 输入框
4. 在图右侧增加 `HistogramLUTWidget` 颜色条
5. 颜色条与 `ImageItem` 直接绑定，支持显示当前图像数据分布

支持的色图包括：

- Jet
- Viridis
- Plasma
- Inferno
- Magma
- Seismic
- Gray
- Hot
- Cool

### 逻辑说明

- 默认情况下，程序仍会根据矩阵数据自动估计一组较稳定的显示范围
- 一旦用户手动修改 `Vmin/Vmax`，后续显示将优先采用手动值
- 颜色条和图像会同步使用同一组 `levels`
- 色图切换会同步作用于主图和颜色条

### 优化效果

- `space-time` 图的对比度可手动调节
- 不同信号幅值下更容易观察特征
- 颜色条提供了实时分布信息，便于判断当前数据动态范围

## 优化项三：DAS Monitoring 按钮状态增强

### 存在问题

原来的 `DAS monitoring` 按钮只是普通按钮：

- 字号偏小
- 视觉反馈较弱
- 启动和停止状态不够直观

### 解决方案

将 `DAS monitoring` 按钮改为带状态样式的可切换按钮，并做如下优化：

- 按钮字号调大
- 增加加粗字体
- 增加明显的背景色状态
- 启动和停止状态显示不同文本

状态逻辑为：

- 未启动时显示 `Start DAS Monitoring`
- 启动后显示 `Stop DAS Monitoring`

### 优化效果

- 按钮状态更直观
- 更容易区分当前监测是否在运行
- 操作反馈明显增强

## 优化项四：增加“图是否更新”的独立控制

### 存在问题

原来的 `Tab3` 没有单独控制图像刷新的开关。若用户只想暂停观察界面刷新，而保持后台 TCP 通信和数据接收继续进行，没有直接入口。

### 解决方案

新增 `Plot Updates` 按钮，用于独立控制 `Tab3` 图形是否刷新：

- `Plot Updates: ON` 时，曲线和 `space-time` 图正常更新
- `Plot Updates: OFF` 时，停止更新 `Tab3` 图形显示
- TCP 通信、DAS 收包和后台处理链路不受影响

### 逻辑说明

关闭绘图更新后：

- 仍继续更新 `Live Header`
- 不再更新 DAS 曲线
- 不再更新 FIP 对比曲线
- 不再更新 `space-time` 图
- 不中断底层通信与数据流

### 优化效果

- 便于在高频数据流下暂停界面刷新
- 有利于减少界面干扰
- 不影响后台数据接收和会话连续性

## 优化项五：联合存储改为按钮形式

### 存在问题

联合存储此前使用复选框：

- 状态提示不够醒目
- 与 `DAS monitoring` 这类关键操作的交互形式不统一

### 解决方案

将联合存储从复选框改为按钮形式，并统一采用与 `DAS monitoring` 相同的按钮风格：

- `Joint Storage: OFF`
- `Joint Storage: ON`

按钮为可切换样式，支持明显的开关状态反馈。

### 兼容性处理

虽然 UI 从复选框改为按钮，但底层设置仍保持布尔开关语义：

- `enabled = button.isChecked()`

因此不会影响原有存储逻辑和 `get_tab3_settings()` 的数据结构。
